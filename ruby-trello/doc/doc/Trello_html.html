    <h1 class="module">module Trello</h1>

<div id="description" class="description">

    <p>Ruby wrapper around the <a href=“Trellotrello.com”>Trello> API</p>

    <p>First, set up your key information. You can get this information by <a href=“clicking”>trello.com/1/appKey/generate“>clicking here</a>.</p>

    <p>You can get the key by going to this url in your browser: <a href=“trello.com/1/authorize?key=TRELLO_CONSUMER_KEY_FROM_ABOVE&name=MyApp&response_type=token&scope=read,write,account&expiration=nevertrello.com/1/authorize?key=TRELLO_CONSUMER_KEY_FROM_ABOVE&name=MyApp&response_type=token&scope=read,write,account&expiration=never”>trello.com/1/authorize?key=TRELLO_CONSUMER_KEY_FROM_ABOVE&name=MyApp&response_type=token&scope=read,write,account&expiration=never> Only request the permissions you need; i.e., scope=read if you only need read, or scope=write if you only need write. Comma separate scopes you need. If you want your token to expire after 30 days, drop the &amp;expiration=never. Then run the following code, where KEY denotes the key returned from the url above:</p>

    <p><a href=“Trello.html#method-c-configure”>::configure</a> do |config|</p>

    <pre>config.consumer_key = TRELLO_CONSUMER_KEY config.consumer_secret = TRELLO_CONSUMER_SECRET config.oauth_token = TRELLO_OAUTH_TOKEN config.oauth_token_secret = TRELLO_OAUTH_TOKEN_SECRET</pre>

    <p>end</p>

    <p>All the calls this library make to <a href=“Trello.html”>Trello</a> require authentication using these keys. Be sure to protect them.</p>

    <p>So lets say you want to get information about the user <strong>bobtester</strong>. We can do something like this:</p>

    <pre>bob = Member.find(&quot;bobtester&quot;) # Print out his name puts bob.full_name # &quot;Bob Tester&quot; # Print his bio puts bob.bio # A wonderfully delightful test user # How about a list of his boards? bob.boards</pre>

    <p>And so much more. Consult the rest of the documentation for more information.</p>

    <p>Feel free to <a href=“peruse”>trello.com/board/ruby-trello/4f092b2ee23cb6fe6d1aaabd“>peruse and participate in our Trello board</a>. It’s completely open to the public.</p>

    </div><!-- description -->
